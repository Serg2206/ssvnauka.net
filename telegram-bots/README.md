# Telegram-боты SSVproff

Пять Telegram-ботов профессора С. В. Сушкова (созданы в @BotFather):

| Бот | Назначение | Переменная с токеном |
|---|---|---|
| @sushkov_medical_bot | О профессоре, направления, консультации | `SUSHKOV_MEDICAL_BOT_TOKEN` |
| @ssvproff_content_bot | Сайты, Telegram-канал, поддержка проекта | `SSVPROFF_CONTENT_BOT_TOKEN` |
| @ssvproff_publisher_bot | Публикации, ORCID / Scholar / Scopus | `SSVPROFF_PUBLISHER_BOT_TOKEN` |
| @ssvproff_youtube_bot | YouTube-канал | `SSVPROFF_YOUTUBE_BOT_TOKEN` |
| @maria_surgical_ai_bot | Ассистент по хирургической тематике | `MARIA_SURGICAL_AI_BOT_TOKEN` |

Без внешних зависимостей — нужен только Node.js 18+ (`npm install` не требуется).

## Быстрый старт

1. **Получите токены.** В Telegram откройте @BotFather → `/mybots` → выберите бота →
   **API Token**. Повторите для каждого из пяти ботов.

2. **Создайте `.env`:**

   ```bash
   cd telegram-bots
   cp .env.example .env
   # вставьте токены в .env
   ```

   Проще всего перечислить все токены в одной переменной `BOT_TOKENS`
   (через запятую) — принадлежность каждого токена своему боту определится
   автоматически. Персональные переменные (`*_BOT_TOKEN`) тоже работают.

   ⚠️ Никогда не публикуйте токены и не коммитьте `.env` (он уже в `.gitignore`).
   Если токен «засветился» — в @BotFather: выбрать бота → API Token → Revoke.

3. **Настройте ботов в Telegram** (имя, описание, меню команд — автоматически):

   ```bash
   npm run setup
   ```

   Скрипт проверяет, что каждый токен принадлежит именно своему боту,
   регистрирует команды и снимает вебхук. Ботов без токена просто пропускает.

4. **Запустите ботов:**

   ```bash
   npm start
   ```

   Все боты с токенами работают в одном процессе через long polling.
   Для постоянной работы разместите на любом сервере/VPS (например, systemd-сервис
   или бесплатный тариф Render/Railway). GitHub Pages для этого не подходит —
   он раздаёт только статические файлы.

## Тесты

```bash
npm test
```

47 тестов: маршрутизация команд, ответы каждого бота, адресация `/cmd@bot` в группах,
лимиты BotFather на описания, клиент Telegram API и скрипт настройки.

## Структура

```
src/core/     — клиент API, роутер сообщений, настройка профиля, загрузка .env
src/bots/     — определения пяти ботов (команды, тексты, ключевые слова)
src/data.js   — общие ссылки и контакты (синхронизированы с ssvnauka.net)
scripts/setup.js — одноразовая настройка профилей в Telegram
test/         — тесты (node:test, запускаются без установки зависимостей)
```

Чтобы изменить тексты ответов — правьте файлы в `src/bots/`, ссылки — `src/data.js`,
затем `npm test` и перезапуск.
